## 一、 C 语言编程导论 (Introduction to C Programming)

### 1. 理论学习

#### 1. 数据类型、控制结构与底层表示
- **基本数据类型与机器字宽**：  
  C 语言是一种静态类型、面向过程的编译型语言。其数据类型的大小依赖于编译器和机器字宽（如在 64 位系统下，`char` 占 1 字节，`int` 占 4 字节，`long` 占 8 字节，指针类型恒占 8 字节）。
- **控制流的底层跳转**：  
  控制流（`if-else`, `while`, `for`, `switch-case`）在编译后，转化为底层的条件跳转指令（如 x86 汇编中的 `je`, `jne`, `jmp` 等）。`switch-case` 语句在分支较多时，编译器会优化生成一张**跳转表（Jump Table）**，实现 $O(1)$ 时间复杂度的多路分支寻址。

#### 2. 指针的本质与多级指针
- **指针即地址**：  
  指针变量存放的是内存中某块数据的首地址。声明 `int *p` 意味着 `p` 保存一个地址，而 `*p` 则是对该地址进行解引用（Dereference）来访问 4 字节的整型空间。
- **多级指针与指针运算**：  
  `int **pp` 是二级指针，存放的是一级指针 `p` 的内存地址。指针运算（如 `p + 1`）在编译时会根据指针指向的类型大小进行缩放：若 `p` 是 `int*`，则 `p + 1` 的实际地址增加 `sizeof(int)` 字节（即 4 字节）。
- **函数指针（Function Pointer）**：  
  定义形如 `void (*func_ptr)(int)`，它存放的是函数代码段的首地址。函数指针是 C 语言实现**回调函数（Callback）**和模拟**面向对象多态性（虚函数表）**的底层技术支撑。

#### 3. 手动内存管理与复合布局
- **堆内存的分配与生命周期**：  
  `malloc` / `calloc` / `realloc` 在堆区申请指定字节的内存，返回 `void*` 指针。使用完毕后，必须显式调用 `free` 释放，否则会造成内存泄漏（Memory Leak）。释放后需将指针置为 `NULL`，防止产生**悬空指针（Dangling Pointer）**或野指针。
- **结构体（struct）与内存对齐（Padding）**：  
  为提高 CPU 访问内存的效率，编译器在安排结构体成员时通常遵循“自然对齐”原则（各数组成员的偏移量必须是其自身大小的整数倍，且结构体总大小必须是最大成员大小的整数倍）。这会在成员间填补多余字节（Padding）。
- **联合体（union）的共享内存特征**：  
  联合体所有成员共享同一块内存首地址，大小等于最大数组成员的大小。在任一时刻，联合体只能有效存储其中一个成员的值。主要用于节省空间或同一数据的多维类型解释。

### 2. 课后习题

#### Q1. 在 64 位 Linux 操作系统中，若定义了如下变量，则指针 `p` 自身所占的字节数是：
```c
double x = 3.14;
double *p = &x;
```
- A. 4
- B. 8
- C. 16
- D. 依编译器类型而定
- **正确答案**：B
- **解析**：在 64 位架构的计算机系统中，所有类型的指针变量（无论是 `char*`、`int*` 还是 `double*`）在内存中都占用 8 个字节（64 位宽的物理寻址地址）。

#### Q2. 设 `int a[5] = {10, 20, 30, 40, 50};`，则表达式 `*(a + 2)` 的值是：
- A. 10
- B. 20
- C. 30
- D. 40
- **正确答案**：C
- **解析**：数组名 `a` 在计算时会衰退为指向首元素的指针，`a + 2` 表示指针向后移动了两个 `int` 大小（8字节）的距离，指向 `a[2]`。对其解引用 `*(a + 2)` 相当于求 `a[2]`，值为 30。

#### Q3. 指针变量在被 `free()` 释放后，如果未将其赋值为 `NULL`，容易引发的安全隐患是：
- A. 内存泄漏（Memory Leak）
- B. 栈溢出（Stack Overflow）
- C. 悬空指针/野指针（Dangling Pointer）引起的非法内存访问
- D. 编译器产生链接错误
- **正确答案**：C
- **解析**：`free(p)` 只释放了指针指向的堆内存，但指针变量 `p` 内部存放的地址数值并未被清除。如果后续再次解引用 `*p` 或进行写操作，将导致悬空指针引起的非法内存访问或双重释放（Double Free）崩溃。

#### Q4. 设有如下结构体定义：
```c
struct Sample {
    char a;
    int b;
    char c;
};
```
在开启默认对齐的 64 位编译器下，`sizeof(struct Sample)` 的值通常是：
- A. 6
- B. 8
- C. 12
- D. 16
- **正确答案**：C
- **解析**：按照对齐规则：
  - `char a` 占用 1 字节（偏移量为 0）。
  - `int b` 占 4 字节，要求偏移量为 4 的倍数，因此在 `a` 后面填充 3 个 Padding 字节，`b` 存放在偏移量 4~7 处。
  - `char c` 占 1 字节，存放在偏移量 8 处。
  - 结构体总大小必须是最大成员（`int` 4字节）的倍数，所以 9 字节需向上对齐至 12 字节（在 `c` 后面填充 3 个 Padding 字节）。

#### Q5. 关于 C 语言中联合体（union）的特征，下列说法正确的是：
- A. 联合体中各个数组成员分别拥有独立的内存空间，互不干扰
- B. 联合体的大小是其所有成员大小之和
- C. 联合体的所有成员共享同一块内存首地址
- D. 联合体只允许在运行时动态分配空间，不能静态声明
- **正确答案**：C
- **解析**：联合体（Union）的设计初衷是让其成员复用同一块内存区域，因此所有成员的起始地址是完全一致的，修改其中一个成员会覆盖其他成员的值。

#### Q6. 函数指针定义为 `int (*p)(char, char);`，下列关于该指针的描述正确的是：
- A. `p` 是一个指向整型数据的指针，并带有两个字符参数
- B. `p` 是一个指向函数的指针，该函数接收两个 `char` 参数并返回一个 `int`
- C. `p` 是一个返回类型为 `int*` 的普通函数
- D. 这是一个非法的 C 语言声明形式
- **正确答案**：B
- **解析**：括号 `(*p)` 说明 `p` 是一个指针变量，后面的 `(char, char)` 说明它指向一个函数，最前方的 `int` 是该函数的返回类型。

#### Q7. C 语言分配堆内存的函数中，`calloc(n, size)` 与 `malloc(n * size)` 的主要区别在于：
- A. `calloc` 分配的内存位于栈区，而 `malloc` 位于堆区
- B. `calloc` 速度更快，但空间受限
- C. `calloc` 会自动将分配出的内存空间全部初始化为零，而 `malloc` 中是未定义的随机值值
- D. `calloc` 不需要使用 `free` 释放
- **正确答案**：C
- **解析**：`malloc` 仅分配物理空间而不作清理，速度快但包含脏数据；`calloc` 不仅分配空间，还负责将内存中的所有比特位自动清零（BSS样初始化）。

#### Q8. 设有定义 `int a = 10; int *p = &a; int **pp = &p;`，则表达式 `**pp` 代表：
- A. 指针变量 `p` 的内存地址
- B. 变量 `a` 的内存地址
- C. 变量 `a` 的值 10
- D. 非法的数据解引用
- **正确答案**：C
- **解析**：`pp` 是指向指针 `p` 的二级指针。第一次解引用 `*pp` 得到指针 `p`，第二次解引用 `**pp` 得到 `p` 指向的变量 `a`，故其值等于 `a` 的值，即 10。

#### Q9. C 语言预处理指令中，宏定义 `#define SQUARE(x) x * x` 在求值 `SQUARE(2 + 3)` 时的结果是：
- A. 25
- B. 11
- C. 15
- D. 13
- **正确答案**：B
- **解析**：宏定义是简单的文本替换。展开后表达式为：`2 + 3 * 2 + 3`。根据乘法优先级，先算 `3 * 2 = 6`，再加上前面的 `2` 和后面的 `3`，得到 `2 + 6 + 3 = 11`。若要得到 25，应定义为 `#define SQUARE(x) ((x) * (x))`。

#### Q10. 在 C 语言中，将外部文件定义的全局变量引入到当前源文件中使用的关键字是：
- A. `static`
- B. `extern`
- C. `register`
- D. `volatile`
- **正确答案**：B
- **解析**：`extern` 关键字用于声明一个在其他文件（或外部）定义的全局变量或函数，告诉编译器在链接阶段再去寻找其实际定义位置。

## 二、 Python 编程 (Python Programming)

### 1. 理论学习

#### 1. 动态类型、控制流与基本数据结构
- **Python 的对象模型与动态类型**：  
  Python 是动态强类型解释型语言。变量在 Python 中仅仅是一个**指向对象的引用（标签）**，对象本身才拥有类型。所有 Python 对象都有一个包含引用计数、类型指针和数值的头部结构（PyObject）。
- **列表与元组的底层实现**：  
  列表（`list`）是动态数组，支持 $O(1)$ 的随机访问，但在扩容时会触发内存拷贝；元组（`tuple`）是只读的静态数组，一旦创建不可修改，具有更高的内存利用效率。

#### 2. 闭包与装饰器、迭代器与生成器
- **闭包（Closure）的形成条件**：  
  当一个嵌套函数引用了其外部函数的局部变量，且外部函数返回了这个嵌套函数时，就形成了一个闭包。闭包保存了外部函数的环境上下文（自由变量）。
- **装饰器（Decorator）的设计模式与语法糖**：  
  装饰器是一个高阶函数，其作用是接收一个函数并返回一个包装后的新函数。通过 `@decorator` 语法糖，可以在不修改目标函数源码的情况下，动态注入日志、统计、权限校验等横切关注点。
- **迭代器（Iterator）与迭代协议**：  
  迭代器是实现了迭代协议的对象，必须具备 `__iter__()`（返回迭代器自身）和 `__next__()`（返回下一个元素，没有元素时抛出 `StopIteration` 异常）方法。
- **生成器（Generator）的惰性求值（Lazy Evaluation）**：  
  在函数中使用 `yield` 关键字即可将其转化为生成器。生成器在调用时不会立即执行代码，而是返回一个生成器迭代器。每次调用 `__next__()`，函数执行到 `yield` 挂起，保留局部变量状态，并返回 `yield` 后面的值。这使得程序能以 $O(1)$ 空间复杂度处理无限流或海量数据集。

#### 3. 并发编程与全局解释器锁（GIL）
- **GIL（Global Interpreter Lock）的本质与影响**：  
  为了保证 CPython 解释器内部引用计数和垃圾回收等内存管理逻辑的线程安全，Python 设计了全局解释器锁。**在任意时刻，只允许一个线程占用 CPython 解释器执行 Python 字节码**。
- **并发方案的选择**：
    - **多线程（`threading`）**：由于 GIL 存在，多线程在 CPU 密集型任务中无法实现真正的多核并行，甚至会因为线程频繁切换降低效率；但在 I/O 密集型任务（网络、读写）中，线程在等待 I/O 时会释放 GIL，因此多线程依旧能带来性能提升。
    - **多进程（`multiprocessing`）**：每个 Python 进程拥有独立的解释器和内存空间（独立的 GIL），因此多进程能够实现多核 CPU 的并行运算，适用于 CPU 密集型任务。
    - **协程与异步 IO（`asyncio`）**：协程是一种用户态的轻量级线程（单线程内合作式多任务）。通过 `async`/`await`，在单线程内遇到 I/O 阻塞时自动切换到其他协程运行，消除系统级线程上下文切换开销。

### 2. 课后习题

#### Q1. Python 中，变量在赋值时的实质是：
- A. 在内存中分配特定字节的空间并将变量值写入
- B. 创建一个对象，并将变量名（引用标签）绑定到该对象上
- C. 将变量转换为对应类型的强类型静态变量
- D. 仅仅进行宏文本替换
- **正确答案**：B
- **解析**：Python 中一切皆对象。变量名本身并不拥有数据类型和固定内存空间，它只是指向堆内存中具体 PyObject 对象的一个引用标签。

#### Q2. 运行如下 Python 代码后，输出的结果是：
```python
def make_multiplier(x):
    def multiplier(y):
        return x * y
    return multiplier

double = make_multiplier(2)
print(double(5))
```
- A. 2
- B. 5
- C. 10
- D. 25
- **正确答案**：C
- **解析**：`make_multiplier` 返回内部函数 `multiplier`，这构成了一个闭包。`double` 保留了自由变量 `x = 2` 的上下文。当执行 `double(5)` 时，实际执行的是 `2 * 5`，返回 10。

#### Q3. Python 装饰器（Decorator）在底层实现时依赖的核心特性是：
- A. 编译器在前端执行的宏展开
- B. 函数是一等公民（First-Class Object），可以作为参数传递并返回
- C. Python 支持类的多继承
- D. CPU 底层硬件提供的特定跳转指令
- **正确答案**：B
- **解析**：Python 装饰器是典型的高阶函数，它利用了 Python 中函数是一等公民的特性，即函数可以像普通对象一样被赋值、作为参数传入另一个函数、或作为另一个函数的返回值。

#### Q4. 关于 Python 生成器（Generator）和 `yield` 关键字，下列叙述错误的是：
- A. 含有 `yield` 的函数在被调用时不会立即执行函数体
- B. 每次调用 `__next__()` 时，生成器在 `yield` 语句处暂停，并保留所有当前局部变量状态
- C. 生成器可以利用惰性求值极大节省处理大型数据集时的内存开销
- D. 生成器必须一次性把所有生成的内容全部载入主内存才能开始迭代
- **正确答案**：D
- **解析**：生成器是惰性求值的代表，它只有在迭代请求下一个元素（调用 `next()`）时才即时计算并吐出一个数据，并不在内存中积存完整数据，因此空间效率极高。

#### Q5. CPython 解释器中的全局解释器锁（GIL）对多线程程序运行的主要限制是：
- A. 阻止 Python 程序访问外部的 I/O 资源
- B. 使得多线程无法在多核 CPU 上实现真正的物理并行，限制了 CPU 密集型任务
- C. 强制所有局部变量必须是只读的
- D. 使得多线程程序无法在 Windows 系统下执行
- **正确答案**：B
- **解析**：GIL 保证同一时刻只有一个线程在解释器中执行，这锁死了 CPU 密集型任务利用多核的能力（同一时间只有一个核在跑 Python 字节码）。但在等待网络/磁盘 I/O 时主线程会释放 GIL，所以 I/O 密集型多线程依然有效。

#### Q6. 在 Python 并发编程中，对于 CPU 密集型运算任务，最合理的并发方案是：
- A. 采用多线程模式（`threading`）
- B. 采用协程与异步 IO 模式（`asyncio`）
- C. 采用多进程模式（`multiprocessing`）
- D. 采用单线程串行模式
- **正确答案**：C
- **解析**：多进程模式会为每个进程开启独立的 Python 解释器（各自拥有独立的 GIL），从而避开了全局锁的限制，能实现真正的多核 CPU 并行计算，适合 CPU 密集任务。

#### Q7. 运行下列 Python 代码后，列表 `a` 和 `b` 的值分别为：
```python
import copy
a = [1, [2, 3]]
b = copy.copy(a)
b[1][0] = 99
```
- A. `a = [1, [2, 3]], b = [1, [99, 3]]`
- B. `a = [1, [99, 3]], b = [1, [99, 3]]`
- C. `a = [1, [99, 3]], b = [1, [2, 3]]`
- D. `a = [1, [2, 3]], b = [1, [2, 3]]`
- **正确答案**：B
- **解析**：`copy.copy()` 执行的是浅拷贝。它只拷贝了外层列表的引用，对于内层嵌套的列表 `[2, 3]`，`a` 和 `b` 指向的是同一个物理对象。因此修改 `b[1][0]` 同样会使 `a[1][0]` 发生改变。

#### Q8. 在 Python 的垃圾回收（GC）机制中，用于解决循环引用对象泄漏的核心算法是：
- A. 引用计数（Reference Counting）
- B. 标记-清除（Mark-Sweep）与分代收集（Generational Collection）
- C. 内存分配池（PyMalloc）
- D. 静态静态空间检查
- **正确答案**：B
- **解析**：Python 的首要 GC 机制是引用计数，但它无法消除循环引用。因此 Python 额外引入了“标记-清除”来检测孤立的环状链表垃圾，并基于“分代回收”策略将其划分为三代（Generation 0, 1, 2）提高扫描效率。

#### Q9. 运行以下 Python 代码，输出的最终结果是：
```python
def add_to(num, target=[]):
    target.append(num)
    return target

print(add_to(1))
print(add_to(2))
```
- A. `[1]` 和 `[2]`
- B. `[1]` 和 `[1, 2]`
- C. `[1, 2]` 和 `[1, 2]`
- D. 会跑出运行时异常
- **正确答案**：B
- **解析**：Python 的默认参数是在**函数定义期（而非运行期）评估并初始化**的。可变对象 `target=[]` 在定义时创建了一次，之后所有未传入 `target` 的调用都将共享这同一个列表对象，故第二次调用时会在原列表后追加 `2`，返回 `[1, 2]`。

#### Q10. Python 3.5 引入的异步协程语法关键字是：
- A. `yield from`
- B. `async` 和 `await`
- C. `def` 和 `yield`
- D. `run` 和 `wait`
- **正确答案**：B
- **解析**：从 Python 3.5 开始，官方正式引入了 `async def` 定义协程函数，以及使用 `await` 挂起阻塞 IO 任务的关键字，使得异步并发代码更具可读性。

## 三、 Java 面向对象编程 (Java Object-Oriented Programming)

### 1. 理论学习

#### 1. 面向对象三要素与多态底层原理
- **三大特性**：封装（隐藏内部实现）、继承（代码复用与层次构建）和多态（同一接口的多维度具体实现）。
- **多态（Polymorphism）在 JVM 中的实现原理**：  
  多态的本质是动态绑定（Dynamic Binding）。Java 字节码调用普通实例方法时，使用 `invokevirtual` 指令。
  *方法表机制*：在 JVM 加载类时，会在方法区为每个类构建一张**方法表（vtable）**，其中按顺序存放着该类所有可调用方法的物理地址。如果子类重写了父类方法，则对应偏移量处的方法指针将被替换为子类实现的地址；如果子类未重写，则指针依然指向父类实现。在运行时，JVM 根据对象的实际类型（通过对象头中的类指针找到 vtable），定位对应偏移量处的方法并执行。这实现了运行时的动态派发。

#### 2. 抽象隔离、反射机制与动态代理
- **接口（Interface）与抽象类（Abstract Class）的解耦作用**：  
  接口定义行为契约，实现完全隔离与解耦；抽象类用于提取公共实现并强制子类完成特定行为。
- **反射机制（Reflection）的实现**：  
  反射允许 Java 程序在运行时动态加载未知的类（通过 `Class.forName()`），获取类的构造器、方法、属性等元数据，并进行动态调用。反射是 Spring（IoC/DI 容器）和 MyBatis 等工业级框架的核心技术支撑。
- **JDK 动态代理机制**：  
  在运行时，基于反射，通过 `Proxy.newProxyInstance()` 动态在内存中生成一个实现了指定接口的代理类字节码并加载入 JVM，使用 `InvocationHandler` 的 `invoke()` 统一拦截目标对象的方法调用并执行前置/后置逻辑。

#### 3. 并发编程与 Java 内存模型（JMM）
- **JMM（Java Memory Model）的物理抽象**：  
  JMM 规范了多线程之间共享变量的可见性、原子性和有序性行为。它将内存划分为全局共享的**主内存（Main Memory）**和每个线程私有的**工作内存（Working Memory）**。线程对变量的读写必须先在自己的工作内存中进行，再同步回主内存。
- **并发三大特征的实现机制**：
    - **可见性（Visibility）**：由 `volatile` 关键字提供保障。当一个变量被声明为 `volatile` 时，JMM 规定：每次使用该变量必须从主内存重新读取，且每次写入后必须立即同步回主内存。同时，`volatile` 可以防止指令重排序。
    - **原子性（Atomicity）**：通过内置锁 `synchronized` 或 JUC 包中的 CAS（Compare-And-Swap，无锁自旋，依赖 CPU 原生原子指令 `cmpxchg`）及显式锁 `ReentrantLock` 保证一段代码的排他性执行。
    - **有序性（Ordering）**：通过 **Happens-Before** 原则（如程序次序规则、锁定规则、volatile 变量规则、传递性规则）对多线程间执行顺序提供偏序约束。

### 2. 课后习题

#### Q1. 在 Java 虚拟机中，动态绑定（Dynamic Binding，即多态）的底层依赖是：
- A. 编译器直接进行的硬编码地址展开
- B. 每个类的方法表（vtable）中方法指针的动态派发
- C. 运行时动态生成的代理类
- D. 堆内存中对象的垃圾回收状态
- **正确答案**：B
- **解析**：Java 方法调用默认是动态绑定的。JVM 会在类加载时为每个类构建方法表（vtable）。执行子类对象方法时，JVM 查找该对象实际类型的方法表，调用对应偏移量处的具体实现地址。

#### Q2. 设有以下 Java 代码，运行后输出的结果是：
```java
class Parent {
    public void show() { System.out.println("Parent"); }
}
class Child extends Parent {
    public void show() { System.out.println("Child"); }
}
public class Test {
    public static void main(String[] args) {
        Parent p = new Child();
        p.show();
    }
}
```
- A. Parent
- B. Child
- C. Parent 和 Child 均输出
- D. 编译报错，无法将子类对象赋给父类引用
- **正确答案**：B
- **解析**：这是经典的多态行为。虽然引用 `p` 的声明类型是 `Parent`，但其指向的实际运行期对象是 `Child`。根据动态绑定原则，执行的是子类重写后的 `show()` 方法，输出 "Child"。

#### Q3. 关于 Java 的接口（Interface）与抽象类（Abstract Class），下列叙述错误的是：
- A. 抽象类中可以定义非抽象的成员方法并包含普通成员变量
- B. 接口中声明的变量默认都是 `public static final` 类型
- C. 一个类可以实现多个接口，但只能继承一个抽象类
- D. 接口和抽象类都可以直接通过 `new` 关键字进行实例化
- **正确答案**：D
- **解析**：接口和抽象类都属于不完整的类型定义，主要是为子类提供结构模板或规范契约，因此绝对不能直接使用 `new` 实例化，必须由具体的实现类来实现其方法。

#### Q4. Java 反射（Reflection）机制赋予程序的能力不包括：
- A. 在运行时获取任意类的 Class 对象并检测其构造方法、属性和方法
- B. 在编译期自动检测并消除代码中所有的 NullPointerException 隐患
- C. 在运行时动态创建类的实例并进行方法调用
- D. 在运行时强行修改 private 修饰的私有属性的可访问性
- **正确答案**：B
- **解析**：反射是在运行时（Runtime）起作用的动态特性，无法在静态编译期解决空指针异常这类逻辑漏洞。但通过 `field.setAccessible(true)` 反射可以越过可访问性检查，强行读写私有属性。

#### Q5. JDK 动态代理（Dynamic Proxy）技术的主要局限性在于：
- A. 无法在运行时动态生成字节码
- B. 目标被代理类必须实现至少一个接口才能被代理
- C. 会使目标方法的执行效率降低数十倍
- D. 代理类仅支持在单线程下执行
- **正确答案**：B
- **解析**：JDK 动态代理利用代理类和目标类实现相同接口来伪装目标类。如果目标类没有实现任何接口，则无法使用 JDK 动态代理（必须求助于 CGLIB 这类基于继承创建子类来实现的代理技术）。

#### Q6. Java 内存模型（JMM）将内存划分为全局的“主内存”和线程独占的“工作内存”。线程对变量的修改：
- A. 可以直接作用在主内存上，从而其他线程立即感知
- B. 必须先写在工作内存中，之后在合适时机同步回主内存
- C. 完全不需要写回主内存
- D. 只能写在寄存器中
- **正确答案**：B
- **解析**：JMM 规定，所有变量存储在主内存中，但每个线程有自己的工作内存。线程修改变量必须先在工作内存中进行，然后同步回主内存。若未做可见性控制，其他线程可能会读取到过期的本地工作内存值。

#### Q7. 在 Java 并发编程中，关键字 `volatile` 能够保证共享变量的：
- A. 原子性与可见性
- B. 原子性与有序性
- C. 可见性与有序性（禁止重排序）
- D. 绝对的线程安全
- **正确答案**：C
- **解析**：`volatile` 能够保证：1. 写入数据立即同步回主内存且强制其他线程读取新值（可见性）；2. 通过内存屏障禁止指令重排序（有序性）。但它无法保证复合操作（如 `i++`）的原子性。

#### Q8. 实现多线程原子操作时，无锁并发机制（CAS）所依赖底层的硬件指令通常是：
- A. `jmp`（无条件跳转）
- B. `cmpxchg`（比较并交换）
- C. `push` / `pop`（栈操作）
- D. `iret`（中断返回）
- **正确答案**：B
- **解析**：CAS (Compare And Swap) 依靠底层的原子指令（在 x86 架构下是 `lock cmpxchg`），它是一条由 CPU 硬件保证原子性的“比较并交换”指令，是无锁自旋并发的核心。

#### Q9. 声明为 `synchronized` 的普通实例方法，其锁定的是：
- A. 当前方法本身
- B. 当前类的 Class 对象
- C. 当前方法被调用时传入的参数
- D. 触发该方法调用的当前实例对象（`this`）
- **正确答案**：D
- **解析**：Java 中锁都是加在具体对象上的。修饰非静态方法时，默认锁的是 `this` 实例对象；修饰静态方法（`static synchronized`）时，锁的是当前类的 Class 类元对象。

#### Q10. 在 Java 虚拟机的垃圾回收中，元空间（Metaspace，取代了永久代 PermGen）被用来存放：
- A. 所有新创建的普通对象实例
- B. 被多线程共享的类元数据、常量池和方法定义
- C. 线程调用的局部变量和方法帧栈
- D. 垃圾回收器的算法日志
- **正确答案**：B
- **解析**：元空间是方法区的实现。从 Java 8 开始，方法区数据被移入本地内存的“元空间”（Metaspace）中，用于存放类的结构信息、常量池、方法定义等，而普通对象实例依然存放在 JVM 堆（Heap）中。
